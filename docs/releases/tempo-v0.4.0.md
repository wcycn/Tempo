# Tempo v0.4.0-beta

Tempo v0.4.0-beta 是面向同学和朋友的小范围 Android 内测版本。

## 本版本重点

本版本重点完善群组多人活动、个人资料与本地日程管理，并补齐模板日程导入和发布工程检查能力。

## 新增与优化

- 群组活动支持最低人数和确定人数两种参与模式。
- 群组活动支持固定时间、最近时间、人数峰值三种匹配规则。
- 群组确认阶段显示具体时间、匹配规则和确认进度。
- 匹配中会自动关闭接龙入口，避免重复触发匹配。
- 群主可以取消确认中的活动，拒绝者在重新匹配时不会被自动重新加入。
- 群组活动显示真实发起人，并支持群名和成员管理。
- “我的”页面支持分类标签新增、删除和拖动排序。
- 支持每日抽签和资料折叠展示。
- 支持 CSV、XLSX、ICS 模板导入。
- 导入前可预览并选择统一分类，导入后作为本地普通日程保存。
- 导入时自动检测格式错误和日程冲突，并生成导入结果报告。
- 模板导入数据按账号保存在 Room，不上传服务器。
- 增加 GitHub Actions 后端测试、Android 编译和敏感配置检查。
- 增加正式发布前的安全、构建、备份检查清单。

## 测试建议

- 使用两个以上账号测试好友、邀约和群组流程。
- 分别测试最低人数和确定人数模式。
- 测试固定时间、最近时间和人数峰值规则。
- 在确认阶段尝试加入、拒绝、超时、重算和取消。
- 导入 CSV、XLSX、ICS 文件，并测试重复日程和格式错误。
- 断网后检查本地日程是否仍可查看、编辑和删除。
- 测试浅色、深色、跟随系统三种主题。

## 已知限制

- 当前仅支持 Android，仍属于内测版本。
- AI 语音功能依赖服务器和第三方语音服务，可能出现网络超时。
- 正式公网使用仍建议配置域名和 HTTPS。
- FCM 推送和外部系统日历同步尚未启用。

欢迎通过 GitHub Issues 提交手机型号、Android 版本、操作步骤、实际结果和截图。
