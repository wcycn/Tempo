# Tempo 日程模板规范

## 推荐格式

Tempo 采用两层兼容策略：

1. `CSV/XLSX`：用于批量导入和普通用户编辑；
2. `ICS/iCalendar`：用于和系统日历、Google Calendar、Outlook 等交换。

其中 iCalendar 的行业标准是 RFC 5545。CSV/XLSX 没有统一的全球格式，因此 Tempo 使用自己的模板列名。

## CSV/XLSX 模板列

| 列名 | 必填 | 示例 |
| --- | --- | --- |
| title | 是 | 产品会议 |
| start | 是 | 2026-07-20 14:00 |
| end | 是 | 2026-07-20 15:00 |
| category | 否 | 工作 |
| status | 否 | HARD |
| description | 否 | 周会 |

导入时没有填写的 `category` 默认使用“工作”，没有填写的 `status` 默认使用硬性事务。导入前进行格式校验和冲突报告。

## 重复日程

重复日程仍然使用 Tempo 创建日程中的重复规则生成本地事件；ICS 导入时当前导入每个 `VEVENT` 为一个普通本地日程。Android 客户端已在“我的 → 数据导入”提供文件选择、解析预览、统一分类和冲突报告。有效记录会作为本地普通日程写入 Room，默认状态为 `HARD`；不会上传到服务器。与已有本地日程或同批次记录重叠的记录会跳过并在结果中报告，文件本身不会被修改。
