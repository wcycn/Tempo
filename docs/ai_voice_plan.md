# Tempo AI 语音创建日程预留方案

## 当前状态

后端已经实现 `POST /api/ai/calendar/parse-audio`。接口需要登录，接收一段短音频和用户时区，返回未保存的 `CalendarDraft`。服务器已配置智谱 Provider；开发环境未配置密钥时仍会返回 503。

Android 流程为：录音 → 上传后端 → 展示识别草稿 → 用户编辑并确认 → 使用现有本地 Room 保存。AI 不会直接写入日历。录音最多 30 秒，客户端和服务器都不持久化音频。

## 智谱接入

默认适配智谱的 `glm-asr-2512` 语音转文字接口和 JSON 结构化输出。服务端环境变量示例：

```env
ZHIPU_API_KEY=只放在服务器环境变量中
ZHIPU_ASR_MODEL=glm-asr-2512
ZHIPU_TEXT_MODEL=glm-4-flash
```

API Key 不得写入 Android、GitHub、README、日志或数据库。默认不保存音频和原始转写文本；若未来需要排查问题，也只应在用户明确同意后短期保存。

## 解析规则

- 没有明确日期或时间时返回 `missing_fields`，由用户补充，不自动猜测。
- “明天”“下周”等相对时间以客户端传来的当前日期和时区解析。
- 低置信度结果必须先进入确认编辑页。
- 音频限制为 25 MB；产品层建议录音不超过 30 秒。
- 接口只返回草稿，不调用事件创建接口。
